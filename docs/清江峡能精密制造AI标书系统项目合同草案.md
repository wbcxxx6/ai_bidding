## 第一条 项目名称

清江峡能精密制造 AI 标书系统建设项目。

## 第二条 项目背景与建设目标

甲方拟建设一套部署于公司内网环境的智能标书编制平台，用于支撑三峡集团及水电装备供应链相关投标业务。系统围绕水轮机叶片、专用螺母、紧固件、金属结构件、检验检测、交付保障、现场服务等制造业投标场景，提供招标文件解析、企业知识库检索、投标章节设计、投标正文生成、Word 排版导出、在线编辑与归档等能力。

乙方根据甲方业务需求，负责完成本项目的软件开发、系统集成、内网部署、功能调试、培训交付及验收支持工作。

## 第三条 项目建设内容

乙方应为甲方建设一套 AI 标书系统，主要包括后端服务、业务接口、AI 生成链路、本地知识库、文档处理模块、在线编辑集成、工作台页面及部署运行环境配置。

系统采用公司内网部署方式运行，核心数据、上传文件、生成文件、知识库向量数据及业务数据库均部署或存储于甲方指定内网服务器或甲方认可的本地化环境中。

## 第四条 功能清单

### 4.1 招标文件上传与管理

1. 支持上传 Word、PDF、TXT 等常见招标文件格式。
2. 支持记录原始文件名称、上传时间、业务编号、处理状态等基础信息。
3. 支持对上传文件进行本地存储，便于后续分析和追溯。

### 4.2 招标文件解析

1. 支持 Word 文档正文提取。
2. 支持 PDF 文档文本提取。
3. 支持将招标文件内容用于 AI 预分析、章节提取和投标文件生成。
4. 支持对复杂文件解析失败场景返回明确错误提示。

### 4.3 AI 预分析

1. 自动提取招标文件中的投标要求、资格要求、技术要求、商务条款和评分标准。
2. 自动生成招标文件概要，包括项目内容、服务范围、交付周期、质量标准、验收要求等。
3. 根据甲方制造业业务背景识别与水电装备供应链相关的响应重点。

### 4.4 章节格式提取

1. 自动识别招标文件中的投标文件格式要求。
2. 自动提取商务响应、技术响应、资质证明、承诺函、表格清单等章节要求。
3. 支持将表格类内容整理为结构化 Markdown 表格格式。

### 4.5 投标章节设计

1. 根据招标要求、评分标准和章节格式自动生成投标文件章节结构。
2. 支持区分普通正文章节与表格类章节。
3. 支持生成二级、三级章节及章节编写说明。
4. 章节结构应覆盖技术、商务、资质、质量、安全、交付、售后、项目管理等常见投标内容。

### 4.6 企业知识库与检索增强

1. 支持将甲方历史资料、制造能力说明、质量体系材料、业绩资料等写入本地知识库。
2. 支持基于 ChromaDB 的本地向量检索。
3. 在生成投标正文时，系统可结合知识库内容进行辅助生成。
4. 知识库数据存储在甲方内网或本地服务器环境中。

### 4.7 投标正文生成

1. 根据章节标题、章节描述、招标文件上下文和企业知识库内容生成投标正文。
2. 输出内容应符合制造业投标文件常用表达，语言专业、严谨、可落地。
3. 支持质量管理、工艺流程、检验检测、交付保障、售后服务、安全环保、项目组织等章节生成。
4. 支持多章节批量生成并自动保存中间结果。

### 4.8 Word 文件生成与排版

1. 支持将生成的 Markdown 投标文件转换为 Word 文档。
2. 支持生成完整投标文件。
3. 支持提供 Word 文件下载。
4. 支持后续按甲方要求优化封面、目录、页眉页脚、字体字号、表格样式等排版规则。

### 4.9 在线编辑集成

1. 集成 ONLYOFFICE 文档服务，实现 Word 文件在线预览和编辑。
2. 支持在线编辑后的文件保存回调。
3. 支持编辑后文件覆盖或归档至系统指定目录。
4. ONLYOFFICE 服务采用甲方内网部署或甲方指定服务器部署方式。

### 4.10 工作台页面

1. 提供内网标书工作台页面。
2. 支持按流程完成“上传招标文件、AI 预分析、提取章节格式、生成章节设计、生成 Word、在线编辑”的业务闭环。
3. 页面文案和业务定位符合甲方水电装备精密制造业务场景。

### 4.11 用户识别与基础数据记录

1. 支持基础用户识别。
2. 支持记录用户与上传文件的关联关系。
3. 支持 SQLite 本地数据库存储业务记录。

### 4.12 系统部署与配置

1. 支持 Python Flask 后端服务部署。
2. 支持本地 ChromaDB 持久化目录部署。
3. 支持 SQLite 数据库部署。
4. 支持 ONLYOFFICE Docker 容器部署。
5. 支持环境变量配置，包括大模型 API Key、ONLYOFFICE JWT 密钥、内网访问地址等。
